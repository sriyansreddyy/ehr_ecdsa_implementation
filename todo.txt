## Revised Chaincode Functions
```
PATIENT MANAGEMENT
  RegisterPatient(patientId, name, age, gender, bloodGroup, contact, address)
  GetPatient(patientId)
  GetPatientVisits(patientId)    ← all visits for a patient

VISIT MANAGEMENT
  OpenVisit(patientId, chiefComplaint)            ← receptionist
  AssignDoctor(visitId, doctorId)                 ← receptionist / admin
  AssignNurse(visitId, nurseId)                   ← receptionist / doctor / admin
  CloseVisit(visitId)                             ← doctor (finalizes clinical)

FORWARDING
  ForwardToNurse(visitId, instructions)           ← doctor
  ForwardToDoctor(visitId, notes)                 ← nurse
  ForwardToLab(visitId, tests, instructions)      ← doctor
  LabResultsBack(visitId, labRequestId)           ← auto when lab approves

CLINICAL
  UpdateDiagnosisNotes(visitId, notes)            ← doctor (any time)
  RecordVitals(visitId, vitalsJson)               ← nurse
  AddCareNote(visitId, note)                      ← nurse
  UpdatePrescription(visitId, medications, instructions) ← doctor

LAB (per labRequest within a visit)
  AcknowledgeLabRequest(visitId, labRequestId)    ← labreceptionist
  SubmitLabResult(visitId, labRequestId, results) ← labtechnician/radiologist
  ApproveLabResult(visitId, labRequestId)         ← labsupervisor

DISCHARGE & RECORDS
  DispenseMedication(visitId, details)            ← pharmacist
  FinalizeRecord(visitId)                         ← medrecordofficer
  DischargePatient(visitId, notes)                ← hospitaladmin

INSURANCE
  SubmitClaim(visitId, claimId, amount)           ← billingofficer
  AuditClaim(visitId, notes)                      ← claimsauditor
  ProcessClaim(visitId, decision, reason)         ← insuranceofficer

PATIENT PORTAL (read-only)
  GetVisit(visitId)
  GetPatientVisits(patientId)
  GetVisitHistory(visitId)       ← full blockchain audit trail
```

## Visit Status State Machine
```
OPEN
  └─► WITH_DOCTOR          (AssignDoctor)
        ├─► WITH_NURSE      (ForwardToNurse)
        │     └─► WITH_DOCTOR (ForwardToDoctor)  ← back and forth freely
        ├─► WITH_LAB        (ForwardToLab)
        │     └─► WITH_DOCTOR (lab results back) ← auto-return
        └─► VISIT_FINALIZED (doctor FinalizeVisit)
              └─► RECORD_FINALIZED (medrecordofficer)
                    └─► CLAIM_SUBMITTED → CLAIM_APPROVED/REJECTED
                          └─► DISCHARGED
```

## Patient Portal
```
Patient Registration in DB:
  { patientId: "PAT-001", username: "john.doe", passwordHash: "..." }

Patient Login Flow:
  1. POST /patient/auth/login  → returns JWT with patientId
  2. GET  /patient/visits       → all visits for this patientId
  3. GET  /patient/visits/:id   → full visit detail
  4. GET  /patient/visits/:id/history → blockchain audit trail

Patient sees:
  - All their visits (date, doctor, diagnosis)
  - Vitals from each visit
  - Lab results
  - Prescriptions
  - Claim status
  - But NOT internal notes between staff
```
