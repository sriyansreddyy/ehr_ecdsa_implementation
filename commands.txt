## 2. The real concern — what's actually in MSP folders

You're right to question this. Here's the critical distinction:
```
organizations/
├── peerOrganizations/
│   ├── hospital.example.com/
│   │   ├── msp/              ← PUBLIC only (cacerts, config.yaml)
│   │   ├── peers/
│   │   │   └── peer0/
│   │   │       ├── msp/      ← peer's own cert + private key  ✓ safe on VM1
│   │   │       └── tls/      ← peer's own TLS cert + key      ✓ safe on VM1
│   │   └── users/
│   │       ├── hospitaladmin/msp/   ← private key             ✓ safe on VM1
│   │       └── doctor/msp/          ← private key             ✓ safe on VM1
│   │
│   └── diagnostic.example.com/
│       ├── msp/              ← PUBLIC only                    ✓ needed on VM1
│       ├── peers/
│       │   └── peer0/
│       │       ├── msp/      ← diag peer's private key        ⚠ sensitive
│       │       └── tls/      ← diag peer's TLS key            ⚠ sensitive
│       └── users/
│           └── labadmin/msp/ ← labadmin's private key         ⚠ sensitive
```
